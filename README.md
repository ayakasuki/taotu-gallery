# 桃图智库 (taotu-gallery)

> 自托管图床 + 智能标签管理平台，支持 AI 标签扩展、条件自动标签、标签分组、参数化 API、多用户隔离。

v0.3.2-pre · [MIT License](LICENSE)

---

## 功能概览

| 模块 | 功能 |
|------|------|
| 图库展示 | 瀑布流 / 网格两种模式，公共图库 / 我的图库 / 用户图库筛选，新版访客端视觉与导航 |
| 标签系统 | 人工标签、条件自动标签、系统标签、互斥标签、AI 标签（预留接口） |
| 标签分组 | 二级分组结构，支持分组/子分组/标签多选叠加筛选 |
| 用户标签 | 每个用户独立私有标签系统，普通用户不可公共化，管理员可维护图片所属用户的私有标签 |
| 参数化 API | 按标签/分组/相册/随机查询，支持 Token 认证和互斥标签参数校验 |
| 嵌入代码 | 源地址/HTML/BBCode/Markdown × 缩略/中等/完整 = 12 种组合 |
| 多用户 | 注册登录、邮箱验证码、人机验证码、忘记密码、用户隔离、公共/私有图片控制、用户配额 |
| 相册管理 | 创建/编辑/删除、公共相册、封面设置、相册内图片管理 |
| 条件标签 | 路径正则/排除/分辨率/横图/竖图/正方图，专用表单配置 |
| 上传 | 文件上传、URL 上传、批量上传、新建私有标签、上传成功图床链接批量复制，新版上传工作台 UI |
| 多选操作 | 图库页面多选图片、批量删除、批量移动到相册 |
| 管理后台 | 概览、图片管理、标签设置、条件标签、站点配置、综合配置、运维监控、用户管理、公告中心 |
| 备份恢复 | 整库导出 + 本地图库真实文件打包，恢复前解析 manifest 并可选择恢复项 |
| 云同步 | WebDAV 同步标签/路径/条件/站点配置索引数据 |
| 统计 | API 调用量、上传活跃、用户活跃、热门图片 |
| 公告通知 | 后台公告发布/草稿/置顶，前台通知胶囊、未读状态、详情弹窗 |
| 网站外观 | 默认出厂背景图、自定义背景图、背景遮罩、模糊度调节、网站 Icon 上传、备案号页脚、Boxicons 图标体系与玻璃拟态主题 |
| 前端显示域名 | 支持 CDN / 公网域名配置，嵌入代码自动使用 |
| 自定义路径 | 外部图库路径导入，支持指定相册和批量打标签 |
| 邮件服务 | SMTP 配置、测试邮件、注册/忘记密码验证码，Redis 2 分钟过期 |

### v0.3.2-pre（用户组、存储策略与上传体验预备功能）

本次提交对比上一提交 `1f2ae3d`，在 `0.3.1` 正式版和 MIT 许可证补齐之后进入 `0.3.2-pre` 功能线：重点补入相册级主页隐藏、用户组上传规则、存储策略管理、NSFW 审核骨架、自定义路径删减开关和上传页管理员目标图库选择。新表和新字段都通过 Knex 迁移自动创建；当前测试库已执行 `npm run migrate` 并检查核心表字段、默认用户组、默认本地策略、用户组回填和图片存储策略回填均完整。

- 相册设置新增“隐藏于主页图库”开关；开启后该相册内图片不会出现在首页公共图库、我的图库和管理员用户图库，适合收纳杂图、图链素材或不希望出现在主页图片墙的相册。
- 该开关只影响首页图库列表与首页来源统计，不影响相册详情、图片详情、随机图 API、图片公开属性和已生成外链访问逻辑。
- 数据库新增 `albums.hide_from_gallery` 字段，启动迁移会为新部署和旧库升级自动补齐，默认关闭以保持旧相册展示行为不变。
- 管理后台“用户与权限”新增用户组与存储策略入口：用户组承接最大文件大小、并发上传、周期上传限额、路径/文件命名规则、中等图尺寸、图片质量、允许格式和 NSFW 审核配置；新注册用户和首个管理员会自动归入默认用户组。
- 新增 `user_groups`、`storage_strategies` 表，并为 `users`、`images` 补充用户组/存储策略/NSFW 状态字段；旧库升级会自动创建“系统默认用户组”和“默认本地存储”，并把已有用户绑定到默认组。
- 默认本地存储已参与真实上传路径；COS、OSS、FTP、SFTP 管理 UI 和配置 API 已预留，上传流未接入时会明确拒绝，避免静默写入错误位置。
- NSFW 审核服务支持上传后同步尝试审核和后台定时补扫 `NULL` 状态；`nsfw_status=true` 的图片会从非管理员公开图库、相册、随机图 API 和媒体直链中隔离。
- 自定义路径新增“管理图片删减”开关：默认关闭时删除扫描入库图片只删数据库和派生缩略图，不动外部原图；开启后才允许后台图片删除同步删除该自定义路径原始文件。已配置路径支持编辑绑定相册、绑定标签、批量公开和删减开关，但路径本身锁定不可改。
- 上传页修复回车新增标签：输入新公共/私有标签名后按 Enter 会加入暂存新标签，上传成功时再创建并绑定。管理员上传页新增“目标图库 / 用户图库 / 目标相册”并排选择，默认“我的图库”，可切换全部相册或指定用户相册；后端同时校验普通用户只能上传到自己的相册。
- 上传结果表新增“存储策略”列，按每张图片实际写入的 `images.storage_strategy_id` 返回策略名称，便于后续用户组或策略变化时确认图片当次使用的存储策略。

### v0.3.1 续更（无感会话切换与 UI 细节精修）

本次提交对比上一提交 `8839c2b`，继续保持 `0.3.1` 正式版大框架不变，集中处理正式版发布后第一轮体验续更：重点把前端路由切换的主动会话验证改为被动接口拦截，避免有效登录状态下反复闪出“需要登录”或“正在验证管理员权限”，同时收口 Boxicons 复选框、后台标签公共开关、统计曲线和删除确认弹窗等细节。

- 前端会话策略改为“本地 JWT 先渲染 + 后端接口被动拦截”：页面切换不再主动调用 `/api/admin/auth/me` 做路由级验权，只有实际请求 API 时后端返回 `401` 或会话失效类 `403`，前端才统一清理 Token、用户缓存和 `taotu_token` Cookie 并提示用户重新登录。
- 默认布局、上传页、仪表盘和后台布局都使用本地有效 JWT/用户缓存先恢复界面状态，避免管理员或普通用户正常会话中切换页面时闪烁登录提示、权限验证卡片或后台空白门禁。
- `useApi` 增加统一被动会话失效处理，`useUiCache` 增加 `readAuthPayload()`，后续新增页面应复用该模式，不要在 `onMounted` 或路由切换里重复主动验证。
- 图库首页、仪表盘、后台图片管理、后台标签管理等复选框统一为 Boxicons `checkbox` / `checkbox-checked`，视觉尺寸统一到 `20px`，不再使用原生伪图标或过小 SVG。
- 后台标签新建/编辑弹窗补回“是否公共”开关；管理员在后台可把自己拥有的 `uXX` 用户私有标签设为公共并持久化，普通用户仪表盘和普通用户 API 仍不能自行公共化，后端 `/api/user-tags/:id` 对非管理员提交 `is_public` 返回 `403`。
- 管理后台概览统计卡片和运维监控 API 调用量图表从折线改为平滑贝塞尔曲线，回落和上升不再直上直下。
- 删除/高风险确认弹窗继续统一走 `ConfirmDeleteDialog`，关闭按钮从突兀绿底改为主题粉底、默认白色 X，hover 时 X 变深色，保持粉色设计语言。
- 清理旧 PNG/SVG 图标占位资源，除图片详情加载占位 GIF 等真实动图资产外，控制类图标全部由 Boxicons 组件渲染。

### v0.3.1-pre-fix4 续更（权限隔离与发布前细节收口）

本次提交对比上一提交 `787bf10`，继续保持 `0.3.1-pre-fix4` 版本号不变，集中处理正式版前最后一批安全、体验和部署痛点：重点补强管理员/普通用户权限隔离、私有媒体直链门禁、相册数字胶囊与分页、人工标签 hover 预览、协议版本展示，以及 WebDAV ESM 兼容启动问题。

- 管理后台权限隔离升级：后台布局默认不渲染管理页面内容，必须通过 `/api/admin/auth/me` 确认为管理员后才显示；普通用户手改 URL 或复用会话访问 `/admin` 会被拦截并返回仪表盘/登录页。
- 后端新增统一管理员中间件 `requireAdmin`，为站点级后台 API 补齐角色鉴权；综合配置、条件标签、数据库、备份、云同步、用户管理、站点配置、平台标签等后台能力不再仅靠前端隐藏。

- 前端登录态同步新增同站 `taotu_token` Cookie，解决 `<img>` 无法携带 Authorization 导致私有缩略图无法授权的问题；过期或清理登录态时会同步移除 Cookie。
- 普通用户与管理员人工标签页新增图片 hover 预览：小图悬停后快速显示固定宽 `320px` 的中等图预览，自动判断上下浮层位置并防止出屏，辅助打标签识图。
- 相册详情页网格分页改为按实际容器列数计算每页数量，避免第一页尾部空出多个图片位但下一页仍有图片；图片数量、浏览数、标签等胶囊统一改为数字越长自动撑宽。
- 自定义路径重新扫描保留已绑定标签语义：旧路径重新扫描遇到已有标签时不再报“请选择已有标签”，会沿用路径记录里的标签，只对新增图片补入标签和派生图。
- 隐私政策、服务条款和帮助中心改为读取公开配置中的站点名与版本信息，避免协议版本写死；公开配置返回 `appVersion` / `legalVersion`。
- WebDAV 云同步修复启动兼容：`webdav` 依赖为 ESM 时不再在 CommonJS 服务顶层 `require()`，改为按需动态 `import()` 并缓存模块，避免 PM2 启动失败。
- 本轮未新增独立数据库表；新增配置仍走 `site_config` JSON 结构，旧部署启动迁移链路无需额外手工操作。

## 快速开始

### 环境要求

- Node.js 18+
- MySQL 5.7+ / MariaDB 10.5+
- Redis 6+（注册验证码、图片人机验证码、忘记密码验证码必需）
- npm

### 安装

```bash
git clone https://github.com/ayakasuki/taotu-gallery.git
cd taotu-gallery

# 安装后端依赖
npm install

# 安装前端依赖
cd client && npm install && cd ..

# 配置环境变量
cp .env.example .env
# 编辑 .env 填入数据库、Redis、JWT 等信息
```

### 配置数据库

如果 MySQL 用户有建库权限，可以只在 `.env` 配置 `DB_NAME`，启动时会自动创建数据库；如果没有建库权限，请先手动建库：

```sql
CREATE DATABASE gallery_index CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'gallery'@'localhost' IDENTIFIED BY 'your_password';
GRANT SELECT, INSERT, UPDATE, DELETE, ALTER, LOCK TABLES, CREATE ON gallery_index.* TO 'gallery'@'localhost';
FLUSH PRIVILEGES;
```

编辑 `.env`：

```env
DB_HOST=localhost
DB_PORT=3306
DB_USER=gallery
DB_PASSWORD=your_password
DB_NAME=gallery_index
JWT_SECRET=your_random_secret_string

# 首次空库启动会自动创建首个管理员
DEFAULT_ADMIN_USERNAME=admin
DEFAULT_ADMIN_PASSWORD=your_initial_admin_password
DEFAULT_ADMIN_EMAIL=admin@example.com

PORT=7067

# Redis 用于注册验证码/图片验证码/忘记密码验证码
REDIS_HOST=127.0.0.1
REDIS_PORT=6379
REDIS_USERNAME=
REDIS_PASSWORD=
REDIS_DB=0
```

### 配置 Redis

Redis 用于图片人机验证码、注册邮箱验证码和忘记密码邮箱验证码。验证码 TTL 默认为 120 秒，重发会覆盖旧验证码。

```env
REDIS_HOST=127.0.0.1
REDIS_PORT=6379
REDIS_USERNAME=
REDIS_PASSWORD=
REDIS_DB=0
```

如果开启“注册需邮箱验证”或使用忘记密码功能，还需要在后台网站配置中填写 SMTP 并通过测试邮件。

### 初始化与启动

```bash
# 构建前端
cd client && npx nuxt generate && cd ..

# 启动
pm2 start pm2/pm2.json
```

启动时会自动检查数据库、执行迁移并创建缺失的表。首次空库会自动创建首个管理员，默认用户名为 `admin`；如果 `.env` 未设置 `DEFAULT_ADMIN_PASSWORD`，系统会生成一次性随机密码并打印到启动日志。

### 数据库迁移与初始化保障

- 新部署只需要配置 `.env`、安装依赖并启动服务；`server/index.js` 会先执行启动自检、数据库连通性检查、自动建库、`db.migrate.latest()` 和首个管理员初始化，再加载 Express app。
- 不要在生产或首次部署流程中执行 `seed:run`。旧 seed 用于早期开发，会清空用户数据；当前项目以迁移 + 首个管理员初始化作为唯一启动入口。
- 当前版本会自动创建/迁移核心表：`images`、`albums`、`tags`、`user_tags`、`image_tags`、`album_tags`、`conditions`、`tag_groups`、`custom_paths`、`site_config`、`api_tokens`、`users`、`upload_logs`、`api_logs`、`site_metrics`、`announcements`、`announcement_reads` 等。
- `conditions.tag_id` 是 v0.3.1 系列补齐的关键字段，用于把条件标签配置稳定关联到真实 `tags.id`；旧库升级时迁移会回填已有条件标签关联。
- `custom_paths.make_public` 用于持久化路径扫描“批量公开”开关；`albums.all_picture_public` 用于记录相册“公开所有图片”按钮状态，`albums.hide_from_gallery` 用于记录相册是否从首页图库隐藏，避免刷新后丢失相册级展示策略。
- `data/gallery`、`data/uploads`、`data/backups` 是运行时目录，需确保 Node 进程有读写权限；`data/config` 仅保留兼容快照，不再作为配置来源。

## 架构特点

### 统一部署

前后端共用同一端口，Express 同时服务 API 和 Nuxt 静态文件。通过 PM2 一键部署。

### 无感会话切换

前端不再在每次路由切换或页面挂载时主动调用 `/api/admin/auth/me` 验证会话；默认布局、上传页、仪表盘和后台布局会先读取本地有效 JWT 与用户缓存恢复界面。真正的会话有效性由后端 API 在请求时校验：当任意需要登录的 API 返回 `401`，或返回未审核/禁用等会话失效类 `403` 时，`useApi` 会触发 `clearAuthSession()`，统一清理本地 Token、用户缓存和同站 `taotu_token` Cookie，并广播 `taotu:auth-invalid` 让页面退出登录态。这样正常有效会话切换页面时不会闪“需要登录”或“正在验证管理员权限”，只有实际接口发现会话过期时才要求重新登录。

### 视觉系统

v0.3.1 起，访客端、普通用户端和管理后台统一到浅色玻璃拟态视觉风格，覆盖默认布局、导航、首页图库、相册、图片详情、上传、API 文档、登录注册、仪表盘和主要后台管理页。控制类图标统一由 `client/components/TaotuIcon.vue` 基于 `@boxicons/vue` 渲染，默认使用线性图标，选中/hover/fill 状态再切换填充图标并继承按钮文字色；后续新增按钮、下拉项、侧边栏、toast、复选框和信息提示不要再恢复 PNG 占位或字符伪图标。全站下拉框统一使用 `TaotuSelect`，菜单宽度等于触发框，采用 `2px` 边框、`8px` 圆角、小字号粗体选项、最多 4 项无滚动和上下自动弹出。前台主导航已改为文字按钮，品牌区和通知/用户区跟随页面主体容器宽度对齐；右上角通知/用户菜单、删除确认和高风险操作统一使用精致玻璃拟态弹窗。复选框统一使用 Boxicons `checkbox` / `checkbox-checked`，主操作尺寸当前为 `20px`；确认删除弹窗关闭按钮使用主题粉底白色 X。人工标签页的小图支持 hover 中等图预览，方便管理员和普通用户在不进入详情页时快速识图打标。

### 所有配置存储在数据库

标签、条件标签、标签分组、自定义路径、网站配置全部存储在数据库，不使用 JSON 配置文件。避免了 JSON 与数据库不一致的问题。

### 图片 URL 哈希化

所有图片通过哈希路径对外访问，不暴露服务器真实路径：
```
原图: /image/2026-06-24/abc123def456.jpg
缩略图: /thumb/2026-06-24/abc123def456.jpg?s=thumb
中等图: /thumb/2026-06-24/abc123def456.jpg?s=medium
```
自定义路径扫描导入的图片不会在外部原始目录中写入派生文件；缩略图和中等图统一落在 `data/gallery/.derived/<源图路径hash>/`。`/thumb` 会优先读取原图同目录 `.thumbs`，再读取派生目录，缺失时按需补生成，最后才回退原图。


### API 内外分离

- `/api/internal/*` — 前端内部使用，返回完整数据
- `/api/*` — 对外 API，不暴露本地路径
- `/api/admin/*` — 管理后台 API

### 权限隔离

| 场景 | 未登录 | 普通用户 | 管理员 |
|------|--------|---------|--------|
| 公共图库 | ✅ 可看 | ✅ 可看 | ✅ 可看 |
| 自己的图片 | — | ✅ 可管理 | ✅ 可管理 |
| 别人的图片 | ❌ | ❌ | ✅ 可看 |
| 管理后台 | ❌ | ❌ | ✅ 可访问 |

权限隔离必须以后端校验为准，前端隐藏按钮只作为体验优化。普通用户即使用 curl、手改 URL、复用 JWT 或访问 hash 媒体直链，也不能访问后台 API、站点级配置、别人的私有相册/图片或私有标签。`/image/*` 与 `/thumb/*` 会同时校验图片公开状态、公共相册、资源所有者、管理员身份或 API Token。

## 标签系统

### 标签类型

| 类型 | 存储 | 说明 |
|------|------|------|
| 公共系统标签 | `tags` 表，`is_public=true` | 所有人可见 |
| 私有系统标签 | `tags` 表，`is_public=false` | 仅管理员可见 |
| 用户私有标签 | `user_tags` 表 | 仅创建者可见，ID 以 `u` 前缀区分，支持私有互斥关系 |
| 虚拟系统标签 | 后端 API 自动筛选 | `__untagged` 未标签、`__tagged` 已标签，不可删除 |
| 条件标签 | `conditions` 表 + `tags` 表 | 自动匹配图片 |

### 互斥标签

不可组合标签可以填写多个互斥 ID，例如 `3,4` 或 `3.4`。保存后系统会自动补齐互斥组关系：如果标签 6 互斥 3 和 4，则 3、4、6 三者都会互相排斥。

图库和 API 接口页会禁用互斥标签；直接拼接 API URL 时，后端会对非法互斥组合返回 `400` JSON。用户私有标签互斥必须使用 `u` 前缀 ID，普通用户只能配置自己拥有的私有标签之间互斥。

### 标签分组

支持二级分组结构，用于大量标签的分类管理：

```
游戏 (主分组)
  ├── 原神 (子分组, sid=1)
  │   ├── 角色A
  │   └── 角色B
  └── 星铁 (子分组, sid=2)
      ├── 角色C
      └── 角色D
规格 (主分组)
  └── 横竖图 (子分组, sid=3)
```

### 条件标签

| 条件类型 | 说明 |
|----------|------|
| 路径正则 | 路径包含指定文字 |
| 路径排除 | 路径不含指定文字 |
| 分辨率 | `1080p 以下 / 1080p / 2K / 4K+` 互斥档位，按长短边区间归类 |
| 横图/竖图/正方图 | 三者自动互斥判定，宽高比 `0.9-1.1` 归为正方图 |
| 横竖比 | 当前用于 1:1 正方图筛选，沿用 `0.9-1.1` 容差 |

**执行时机：** 上传后立即执行 / 路径扫描后执行 / 条件创建后立即执行 / 定时扫描 / 手动触发。定时扫描默认增量跳过已拥有对应条件标签的图片；只有手动“立即扫描所有图片”会强制重扫全部。

## 账号与邮件验证

### 注册与找回密码

- 管理员在网站配置中开启开放注册后，注册页通过公开配置接口显示注册表单。
- 开启“注册需邮箱验证”后，注册必须先通过图片人机验证码，再发送邮箱验证码。
- 邮箱验证码固定为 5 位：2 个数字 + 3 个英文乱序组合，Redis TTL 为 120 秒。
- 忘记密码使用账号绑定邮箱发送验证码，重置密码时需要邮箱验证码和新的图片人机验证码。
- SMTP 配置、备案号、公开域名等均保存到 `site_config` 表。

## API 说明

### 认证方式（二选一）

```bash
# 方式一：Authorization Header
curl -H "Authorization: Bearer YOUR_TOKEN" \
     "https://your-domain.com/api/images/random?count=1&pic=md"

# 方式二：URL 参数
curl "https://your-domain.com/api/images/random?count=1&pic=md&tk=YOUR_TOKEN"
```

### 随机图片参数

| 参数 | 说明 |
|------|------|
| `count` | 返回数量，1 返回图片二进制，>1 返回 JSON |
| `tags` | 标签 ID，逗号分隔 |
| `tag_g` | 分组 ID，逗号分隔（返回分组内所有标签的图片） |
| `sid` | 子分组 sid，逗号分隔（返回子分组内标签的图片） |
| `album` | 相册 ID |
| `pic` | `md` 返回中等尺寸图片（减少带宽） |
| `tk` | API Token（URL 参数方式） |

**叠加筛选示例：**
- `?tag_g=1&sid=3&tags=4,5` — 分组1 + 子分组3 + 标签4、5 叠加
- `?tags=1,2,3` — 多个标签叠加
- `?count=1&pic=md` — 返回一张中等尺寸随机图片

若 `tags` 或分组展开后的标签包含互斥组合，API 返回 `400`，例如：

```json
{ "error": "非法互斥标签参数：test14 与 test15 不可同时筛选" }
```

```bash
pm2 start pm2/pm2.json

```

### Nginx 反向代理

```nginx
server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass http://127.0.0.1:7067;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        client_max_body_size 100m;
    }
}
```

### HTTPS

管理后台 → 网站配置 → HTTPS/SSL，填入证书路径和密钥路径，重启服务生效。
